pylibschc.fragmenter¶

User-facing fragmenter/reassembler functionality

class pylibschc.fragmenter.FragmenterReassembler(device: Device, mode: FragmentationMode | None = None, end_rx: Callable[[FragmentationConnection], None] | None = None, end_tx: Callable[[FragmentationConnection], None] | None = None, post_timer_task: Callable[[FragmentationConnection, Callable[[object], None], float, object], None] | None = None, remove_timer_entry: Callable[[FragmentationConnection], None] | None = None)[source]¶

Bases: FragmenterOps

A handler for fragmentation and reassembly.

Warning

If you fragment and reassemble a packet on the same device, you need two objects of this type.

Parameters:

  • device – The device to use for fragmentation/reassembly.

  • mode – (optional) The pylibschc.libschc.FragmentationMode to use.

  • end_rx – (optional) The callback that is called when the reception of a packet is complete.

  • end_tx – (optional) Callback that is called when the transmission of a packet is complete.

  • post_timer_task – (optional) Callback that is called when a timer task needs to be scheduled.

  • remove_timer_entry – (optional) Callback that is called when a timer task needs to be canceled. May be None.

input(data: bytes | BitArray) → ReassemblyStatus[source]¶

Handle incoming data.

Parameters:

data – Either an ACK, a fragment, or an unfragmented packet.

Returns:

Status of reassembly or an ACK was handled.

Raises:

MemoryError – If memory for fragment reception could not be allocated.

Retval MIC_INCORRECT:

when MIC was incorrect in received fragment.

Retval ACK_HANDLED:

when the ACK was handled.

Retval ONGOING:

If reassembly is still missing fragments.

Retval COMPLETED:

If the fragment handled was the last missing fragment (or if the data was not fragmented).

Retval STAY_ALIVE:

If reassembly was completed, but the connection still is kept open, e.g., in case another ACK needs to be sent.
